# 设计说明

## 分层

- `data.js`：九宫、八门、九星、八神、三奇六仪、节气局数表等常量。
- `calendar.js`：历法层，负责四柱、节气、阴阳遁、三元、局数。
- `qimen-core.js`：排盘核心，生成 `chart` 对象。
- `relations.js`：关系层，生成关系边、状态修正项、六维量化、取用索引。
- `export.js`：大模型导出，复制完整盘面与本宫详情。
- `render.js`：UI 渲染。
- `app.js`：事件绑定与应用状态。

## 从 liuyao-graph 继承的模式

`输入 → 历法环境 → 节点 → 关系边 → 状态修正 → 量化 → 卡片展示 → 结构化导出`。

奇门的节点以九宫为主，宫内挂载门、星、神、天盘干、地盘干。关系边用于说明“为什么这个宫受助或受制”，修正项用于说明空亡、门迫、马星、伏吟、干格等非普通生克因素。

## 给大模型的导出

导出不只是九宫表，而是包含：基础信息、取用体系、九宫总览、全部关系边、全部修正项、建议推理顺序、JSON 结构。这样大模型可以基于盘面事实做更有逻辑的推理，而不是凭空套断语。
